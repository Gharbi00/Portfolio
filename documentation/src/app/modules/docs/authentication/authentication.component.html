Authentication

Introduction

Elevok's API employs API keys for request authentication, providing a secure mechanism to access resources. Your API keys can be accessed and managed within the Elevok Dashboard.

Authentication for Elevok APIs is conducted using API keys, which serve as a crucial security measure for your account. Elevok provides two types of API keys: the Public Key and the Secret Key (also referred to as the transaction key).

This API is used to authenticate users and obtain an access token, which is required to access other APIs within the Elevok Loyalty Program system.

Endpoint

URL: /cauth/clogin
Method: POST

Request Parameters

publicKey: Public key associated with the user's account.
secretKey: Secret key associated with the user's account.
client: Your client identifier.

Error Handling

The API returns appropriate error responses in case of missing parameters or authentication failure.

400 Bad Request: If the publicKey or secretKey parameters are missing.
400 Bad Request: If no user is found with the provided email.
400 Bad Request: If the provided secret key is incorrect.

Sample Request

Upon successful authentication, the API returns an access token.


  {
    "publicKey": "example_public_key",
    "secretKey": "example_secret_key"
    "client": "example_client_key"
  }

Sample Response

Upon successful authentication, the API returns an access token.


  {
    "expiresIn": 36000000"
    "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  }
