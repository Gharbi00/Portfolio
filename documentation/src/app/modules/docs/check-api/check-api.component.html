Check User Wallets and Reputation

Introduction

This endpoint offers a comprehensive snapshot of a user's engagement within the platform. It retrieves detailed information regarding the user's various wallets, including their quantitative and qualitative points balances. Additionally, it provides insight into the user's reputation score, reflecting their overall activity and loyalty. This data empowers administrators to assess user engagement levels, tailor rewards, and optimize retention strategies effectively.

Endpoint

URL: /wallet/check
Method: POST

Request Headers

Here you need to insert the bearer token obtained from the Login and Get Access Token API.


  {
    "Authorization": "Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  }

Request Parameters

user: User's unique identifier.
card: Loyalty card identifier.
referral: Referral reference.

Note: Only one of the parameters, either user or card, is required. referral is optional, pass it only when it's available

Error Handling

The API returns appropriate error responses in case the user's card is not found or if no wallets are found for the provided user.

404 Not Found: If the user's card is not found.
404 Not Found: If no wallets are found for the provided user.
400 Bad Request: Reference incorrect!
400 Bad Request: You already redeemed this reference.
400 Bad Request: Someone else already redeemed this reference.

Sample Request

Upon successful authentication, the API returns an access token.


  POST /wallet/check
  Authorization: Bearer YOUR_ACCESS_TOKEN
  {
    "card": "loyalty_card_number"
  }
