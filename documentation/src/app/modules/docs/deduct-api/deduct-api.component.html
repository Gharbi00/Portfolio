Deduct Points from User

Introduction

This endpoint facilitates the deduction of points from a user's account, providing administrators with the ability to enforce consequences or manage reward redemptions effectively. Whether mitigating rule violations, implementing penalty systems, or facilitating reward redemption transactions, this functionality ensures precise control over point allocations. By deducting points when necessary, administrators can maintain fairness, uphold accountability, and cultivate a balanced rewards ecosystem.

Endpoint

URL: /wallet/deduct
Method: POST

Request Headers

Here you need to insert the bearer token obtained from the Login and Get Access Token API.


  {
    "Authorization": "Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  }

Request Parameters

user: User's unique identifier.
card: Loyalty card identifier.
amount: Total amount of the coins you need to deduct.

Note: Only one of the parameters, either user or card, is required.

Error Handling

The API returns appropriate error responses in case the user's card is not found or if no wallets are found for the provided user.

404 Not Found: If the user's card is not found.

Sample Request

Upon successful authentication, the API returns an access token.


  POST /wallet/deduct
  Authorization: Bearer YOUR_ACCESS_TOKEN
  {
    "card": "loyalty_card_number",
    "amount": 87
  }

Sample Response

Upon successful authentication, the API returns an access token.


  {
    "status": 200,
    "response": "Amount was deducted successfully."
  }
