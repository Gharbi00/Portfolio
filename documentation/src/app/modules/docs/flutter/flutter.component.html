Flutter SDK Documentation

Getting Started

The Elevok Widget is a versatile tool designed to enhance user interaction within your application. It offers a comprehensive range of features aimed at optimizing engagement and functionality. Whether you're looking to track user loyalty, streamline profile management, or gain valuable insights into user reputation and achievements, the Elevok Widget delivers. Its stackable bubble design seamlessly integrates into your app's interface, providing a non-intrusive user experience. Moreover, you can customize its appearance to align with your app's branding, ensuring a cohesive and visually appealing integration. With the Elevok Widget, you empower users to engage more deeply with your app while maintaining a consistent and polished user interface.

Install the Elevok Widget Flutter SDK in Your App. The Elevok Widget SDK for Flutter allows you to display the Elevok Widget customer profile in your app, track customer events, and integrate referrals seamlessly.

Installation

Follow these steps to install the Flutter SDK into your app. Use this package as a library.

Run the following command :

flutter pub add flutter_elevok_widget

This will add a line like this to your package's `pubspec.yaml` file and automatically run `flutter pub get` : 

dependencies :
  flutter_elevok_widget : ^x.x.x

Alternatively, your editor might support `flutter pub get`. Check your editor's documentation to learn more.

Now, in your Dart code, you can import the package :

import 'package :flutter_elevok_widget/flutter_elevok_widget.dart';

Integration

To integrate the ElevokWidget into your app, follow these steps :

  • Import the ElevokWidget package at the beginning of your Dart file :
    • import 'package :flutter_elevok_widget/flutter_elevok_widget.dart';
  • Wrap your `MaterialApp` with the `ElevokWidget` widget, configuring it with the necessary parameters :
    •         
  ElevokWidget(
    themeMode : ThemeMode.system,
    locale : const Locale('fr', 'FR'),
    signInRedirection : (BuildContext buildContext) {
      // Note that, If you're using something that requires a BuildContext, you should use the BuildContext returned from our function.
      // Redirect the user to the sign-in view, for example  :
      Navigator.push(context, MaterialPageRoute(builder : (context) => const SignIn()));
    }
    child : MaterialApp(
      // Your MaterialApp configuration goes here
    ),
  );

Now, the ElevokWidget will be integrated into your app, allowing it to be displayed seamlessly across all screens :

Parameters :

child (Widget) : Main app. eg. Material App. (Mandatory)
themeMode (ThemeMode) : Theme mode, eg ThemeMode.system or ThemeMode.dark or ThemeMode.light. (Optional)
locale (Locale) : App locale with this format eg. Locale(‘en’,’US’). (Optional)
signInRedirection (BuildContext buildContext) : Callback function triggered when the login button in the widget is tapped. (Optional)

Note : Fields marked with * are mandatory.

Authentication

For integrating the Elevok Sign-In Widget into your application, you have two modes : full user mode and guest user mode.

  • Full User Mode. In this mode, users provide their email, first name, last name, and optionally, their phone number. These details help personalize their experience within the app. Example :
    •         
  Final ElevokLoginResponse result  = await ElevokFunctions.signInWidget(
    publicKey : 'xxxxxxxxxxxx',
    secretKey : 'xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx',
    appID : 'xxxxxxxxxxxxxxxxxxxxxxxx',
    identifier : 'xxxxxxxxxxxxxxxxxxxxx', // Unique identifier for the user
    email : 'johndoe@example.com',
    firstName : 'John',
    lastName : 'Doe',
    phoneNumber : '123 456 7891',
    phoneNumberCountryCode : '+43',
  );
  • Guest User Mode. Guest user mode is simpler, requiring only the essential authentication keys and app ID. This mode allows users to access basic functionalities of the app without providing personal information. Example : 
    •         
  Final ElevokLoginResponse result = await ElevokFunctions.signInWidget(
    publicKey : 'xxxxxxxxxxxx',
    secretKey : 'xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx',
    appID : 'xxxxxxxxxxxxxxxxxxxxxxxx',
  );

Choose the mode that best suits your application's requirements, whether it's providing a personalized experience for registered users or allowing quick access for guests.

Here are the different responses you can expect from the Elevok login function, along with their descriptions :

  • Incorrect Credentials :
    • Response : Incorrect credentials. Please double-check your login information and try again. If the issue persists, reach out to our support team for assistance.
    Code : 0
    Success : false
  • Login Successful (Guest User) :
    • Response : Login successful! You are now logged in as a guest user.
    Code : 1
    Success : true
  • Unknown Error (Guest User Login) :
    • Response : An unknown error occurred while attempting to log in as a guest user. Please try again, and if the problem persists, contact our support team for help.
    Code : 2
    Success : false
  • Unknown Error (Full User Login) :
    • Response : An unknown error occurred while attempting to log in as a full user. Please try again, and if the issue persists, contact our support team for assistance.
    Code : 3
    Success : false
  • Login Successful (Full User) :
    • Response : Login successful! You are now logged in as a full user.
    Code : 4
    Success : true
  • Unknown Error (Full User Login, Retry) :
    • Response : An unknown error occurred while attempting to log in as a full user. Please try again, and if the issue persists, contact our support team for assistance.
    Code : 5
    Success : false
  • General Unknown Error :
    • Response : An unknown error occurred. Please contact our support team for further assistance.
    Code : 6
    Success : false

These responses provide clear feedback for handling login attempts, ensuring users are informed of any issues encountered during the login process and guiding them on further actions if needed.

Each time the Elevok Widget is initialized with a new Identifier, Email, and Phone Number, it automatically creates or updates the corresponding profile information on its side. This ensures that user data remains current and synchronized with the application's database.

Selecting an unchangeable unique identifier for the Elevok Widget is crucial for creating and maintaining profile. Using a unique Identifier ensures stability in tracking and retaining customer data, such as points and rewards, without the risk of data loss. It's advised against using mutable identifiers like email addresses or mobile numbers, which can be updated by customers, potentially leading to data discrepancies or loss of accumulated rewards and points within Elevok Widget.

Wallet Check

The checkUserWalletAndReputation function retrieves and provides detailed information about the user's wallet and current reputation level within the Elevok Widget.

      
  final result = await ElevokFunctions.checkUserWalletAndReputation();

The function returns an instance of ElevokCheckWalletAndReputationResponse, which contains :

  • Wallet Information : Details about the user's wallet, including the amount of coins, wallet type, and coin details.
  • Current Reputation : Information about the user's current reputation level, including associated perks.

The following section provides a detailed explanation of various Flutter classes used within the Elevok Widget Check scope :

  • Class `Wallet` represents details about the user's wallet.
    • amount : Amount of coins in the user's wallet.
    walletType : Type of wallet, represented using `WalletType` enumeration.
    coin : Details of the specific coin in the wallet, represented by the `Coin` class.
  • Class `Coin` represents details of a specific type of coin.
    • id : Identifier for the coin.
    name : Name of the coin.
    picture : Image associated with the coin, represented by the `Picture` class.
    unitValue : Value of the coin in terms of units, represented by the `UnitValue` class.
  • Class `Picture` represents an image associated with a coin.
    • baseUrl : Base URL for the image.
    path : Path to the image file.
  • Class `UnitValue` represents the value of a coin in terms of units.
    • amount : Amount or value of the coin.
    currency : Instance of `Currency` class representing the currency details.
  • Class `Currency` represents details about a currency.
    • flagSquare : Image of the currency's square flag, represented by the `Picture` class.
    iconFlag : Icon representing the currency's flag.
    name : Name of the currency.
    code : Code representing the currency.
  • Class `CurrentReputation` represents the user's current reputation level.
    • reputationLevel : Current level of reputation for the user.
    color : Color representation associated with the reputation level.
    perks : Instance of `Perks` class providing additional perks or benefits associated with the reputation level.
  • Class `Perks` represents additional perks or benefits associated with a reputation level.
    • description : Description of the perks.
    discount : Discount associated with the perks.

These classes collectively manage and provide structured access to wallet and reputation information for users interacting with the Elevok Widget in your Flutter application. Each class encapsulates specific data attributes and relationships to facilitate efficient handling and presentation of user-related data.

Wallet Push

The pushAmountToUser function allows pushing an amount to the user's wallet. Optionally, it deducts coins from the user's wallet if specified. It needs to be used in this way :

      
  final result = await ElevokFunctions.pushAmountToUser(
    amount : _amount,
    deductAmount : _coins, // Optional, used when redeeming points
    transactionId : 'xxxxxxxxxxx', // Optional external identifier
  );

Parameters :

  • amount : The amount to push to the user's wallet (mandatory).
  • deductAmount : The amount of coins to deduct from the user's wallet (optional). Use this when redeeming points for a purchase.
  • transactionId : Optional external identifier for tracking purposes & To ensure transaction uniqueness.

Note : If deductAmount is provided, it subtracts the specified coins from the user's wallet after pushing the amount. Use transactionId to associate the transaction with an external system or identifier.

Wallet Deduct

The deductCoinsFromUser function deducts a specified amount of coins from the user's wallet. It needs to be used in this way :

      
  final result = await ElevokFunctions.deductCoinsFromUser(
    amount : _coins,
  );

Parameters :

  • amount : The amount of coins to deduct from the user's wallet (mandatory).

Note : Use this function separately when deducting coins from the user's wallet without pushing an amount.

These functions provide flexible ways to manage user wallet transactions within the Elevok Widget, allowing for seamless integration of point redemption and wallet management functionalities in your Flutter application.

User Signout

The signOutWidget function is used to sign out a user from the Elevok Widget and hide the widget from the application interface. It needs to be used in this way :

      
  ElevokFunctions.signOutWidget();

Note : Call this function when you want to sign out a user from the Elevok Widget and hide the widget from the application interface. It ensures that the user's session and associated data within the Elevok Widget are properly cleared and managed.

Use signOutWidget to seamlessly manage user sessions and ensure a smooth sign-out process within your Flutter application integrated with the Elevok Widget.

Animated Buttons

To animate buttons or any type of widget in your application, simply wrap the widget with ElevokAnimatedButton. The activityID parameter is mandatory to uniquely identify the activity. To enable it you need to wrap your widget with ElevokAnimatedButton to make it animated. The activityID argument is required. 

      
  ElevokAnimatedButton(
    activityID : 'xxxxxxxxxxxxxxxxxxxxxxxx',
    child : YourWidget(),
  );

The ElevokAnimatedButton also supports sharing links on social media platforms, such as Facebook (Now, we only support facebook share). For this, use the elevokSocialMediaData parameter to specify the link and an optional hashtag. To share a link on Facebook, provide the link (mandatory) and an optional hashtag using ElevokSocialMediaData. Example : 

      
  ElevokAnimatedButton(
    activityID : 'xxxxxxxxxxxxxxxxxxxxxxxx',
    elevokSocialMediaData : ElevokSocialMediaData(
      link : 'https ://example.com/photoID=0293480923OJ4KL', // Mandatory link
      hashtag : '#bestSellerOfTheWeek', // Optional hashtag
    ),
    child : YourWidget(),
  );

Parameters :

  • activityID : A unique identifier for the animated activity (mandatory).
  • elevokSocialMediaData : Contains social media sharing details (mandatory if the activity is share post on Facebook).
  • link : The URL to be shared on social media (mandatory).
  • hashtag : A hashtag to be included with the shared link (optional).
  • child : The widget to be animated and displayed (mandatory).

Note : Ensure activityID is unique to guarantee uniqueness in transactions and activities. The link parameter in ElevokSocialMediaData is mandatory for social media sharing. The hashtag parameter is optional but can enhance the visibility of your shared content on social media.

Google Maps Integration: Android & IOS

To integrate Google Maps in your Flutter application for both Android and iOS, Run this command:

      
  $ flutter pub add google_maps_flutter

Android

  • Add Permissions: Ensure the necessary permissions are added in your AndroidManifest.xml file
    •         
  // ADD THIS INSIDE <manifest>
  <uses-permission android:name="android.permission.ACCESS_COARSE_LOCATION" />
  <uses-permission android:name="android.permission.ACCESS_FINE_LOCATION" />
  <uses-permission android:name="android.permission.INTERNET" />
  • Add Meta-Data for API Key: Include the Google Maps API key in your AndroidManifest.xml file:
    •         
  // ADD THIS INSIDE <application>
  <meta-data
  android:name="com.google.android.geo.API_KEY"
  android:value="YOUR_GOOGLE_MAPS_API_KEY" />

iOS

  • Add Google Maps Dependency: Add Google Maps dependency in your AppDelegate.swift:
    •         
  // ADD THIS IMPORT AT THE END OF THE FILE
  import GoogleMaps
  // ADD THIS LINE ABOVE ""GeneratedPluginRegistrant.register(with: self)""
  GMSServices.provideAPIKey("YOUR_GOOGLE_MAPS_API_KEY")
  • Update Info.plist: Ensure the following entries are included in your Info.plist: 
    •         
  <key>NSLocationWhenInUseUsageDescription</key>
  <string>This allows you to send locations in chat.</string>
  <key>NSLocationAlwaysUsageDescription</key>
  <string>This allows you to send locations in chat.</string>
  <key>NSLocationAlwaysAndWhenInUseUsageDescription</key>
  <string>This allows you to send locations in chat.</string>
