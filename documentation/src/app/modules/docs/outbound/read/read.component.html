Acknowledge outbound message

Introduction

This endpoint creates a corporate notification linked to an outbound action for a specific target extracted from the authorization token to prevent the outbound to show again.

Endpoint

URL: /outbound/read
Method: POST

Request Headers

Here you need to insert the bearer token obtained from the Login and Get Access Token API.


  {
    "Authorization": "Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  }

Request Parameters

outbound: Outbound identifier.
reference*: Session reference found in local storage or cookies under key elevkusr.

Note: In case no reference is found in cookies, please save the outbound identifier in the cookies to prevent the outbound to show again and before showing any outbount test if you have it already in your cookies.

Error Handling

500 Internal server error: In case of an error (HTTP 500), you need to get in touche with our developers.

Sample Request

Upon successful request, the API returns an success message.


  POST /outbound/read
  Authorization: Bearer YOUR_ACCESS_TOKEN
  {
    "outbound": "sdfgsd9867s987df6g80987"
    "reference": "345635jkbnjkhb34563456"
  }

Sample Response

Upon successful request, the API returns a confirmation.


  {
    "success": "true",
    "message": "Outbound acknowledgment saved successfully!",
  }
