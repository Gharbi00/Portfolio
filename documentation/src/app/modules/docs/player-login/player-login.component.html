Player Login API

Introduction

The Player API in Elevok allows you to create, update, and display player information, as well as attach or detach tags. Players are uniquely identified by a playerUniqueId. This API provides comprehensive control over managing player data within the Elevok platform.

Endpoint

URL: /cauth/login
Method: POST

Request Headers

Here you need to insert the bearer token obtained from the Login and Get Access Token API.


  {
    "Authorization": "Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  }

Request Parameters

identifier*: Unique user ID on the client side.
reference*: Session reference found in local storate or cookies under key elvkwdigtref.
email: User's email address.
firstName: User's first name.
lastName: User's last name.
dateOfBirth: User's date of birth.
username: User's username.
phoneCode: Country code for the user's phone number.
phoneNumber: User's phone number.
referral: Referral code if existing.
gender: User's gender. Must be one of the following:
        - MALE
        - FEMALE
        - PREFER_NOT_TO_SAY
      
maritalStatus: User's marital status. Must be one of the following:
        - SINGLE
        - MARRIED
        - ENGAGED
        - WIDOWED
        - DIVORCED
        - SEPERATED
        - IN_RELATIONSHIP
        - PREFER_NOT_TO_SAY

Error Handling

The API returns appropriate error responses in case of missing parameters or if the target user is not found in the database.404 Not Found: If the target user is not found in the database.

404 Not Found: If the target user is not found in the database.

Sample Request

Upon successful authentication, the API returns an access token.


  POST /cauth/login
  Authorization: Bearer YOUR_ACCESS_TOKEN
  {
    "identifier": "unique_user_identifier",
    "email": "user@example.com",
    "firstName": "John",
    "lastName": "Doe"
  }

Sample Response

Upon successful authentication, the API returns an access token.


  {
    "expiresIn": "360000000"
    "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  }
