Player Logout API

Introduction

The logout endpoint facilitates user session management in real-time. It allows users to transition from their current session into guest mode instantly. This feature ensures smooth user experience transitions and session security.

By invoking the logout endpoint, users can securely end their current session, enabling quick access to guest functionalities or initiating a new session as needed. This real-time functionality enhances user control and privacy within the Elevok platform.

Endpoint

URL: /cauth/logout
Method: POST

Request Headers

Here you need to insert the bearer token obtained from the Login and Get Access Token API.


  {
    "Authorization": "Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  }

Request Parameters

identifier*: Unique user ID on the client side.
reference*: Session reference found in local storate or cookies under key elvkwdigtref.

Error Handling

The API returns appropriate error responses in case of missing parameters or if the target user is not found in the database.404 Not Found: If the target user is not found in the database.

404 Not Found: If the target user is not found in the database.

Sample Request

Upon successful authentication, the API returns an access token.


  POST /cauth/logout
  Authorization: Bearer YOUR_ACCESS_TOKEN
  {
    "identifier": "unique_user_identifier",
    "reference": "fdasf5412adf",
  }

Sample Response

Upon successful logout, the API returns a response.


  {
    "success": "true"
    "message": "Logged out successfully"
  }
