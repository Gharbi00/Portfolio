Push Points to User

Introduction

With this endpoint, administrators can dynamically enhance user engagement by injecting points directly into a user's account. Whether incentivizing specific actions, acknowledging achievements, or reinforcing positive behaviors, this functionality enables seamless point allocation to drive desired user behaviors. By pushing points programmatically, administrators can execute targeted engagement initiatives and foster deeper connections with their user base.

Endpoint

URL: /wallet/push
Method: POST

Request Headers

Here you need to insert the bearer token obtained from the Login and Get Access Token API.


  {
    "Authorization": "Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  }

Request Parameters

user: User's unique identifier.
card: Loyalty card identifier.
amount: Total amount of the purchase.
transactionId: Unique reference of the purchase.
deductAmount: Amount of points to deduct.
source: Source of the transaction must me be one of the following.
        - WEB
        - MOBILE
        - PHYSICAL

Note: Only one of the parameters, either user or card, is required. transactionId is optional, used only when you want to track unicity of your transactions. deductAmount is optional, used only when you need to deduct points when pushing.

Note: if you don't pass source, PHYSICAL will be the deffault value.

Error Handling

The API returns appropriate error responses in case the user's card is not found or if no wallets are found for the provided user.

404 Not Found: If the user's card is not found.
404 Not Acceptable: If the cashback is not active.

Sample Request

Upon successful authentication, the API returns an access token.


  POST /wallet/push
  Authorization: Bearer YOUR_ACCESS_TOKEN
  {
    "card": "loyalty_card_number",
    "amount": 154.84,
    "deductAmount": 45,
    "transactionId": "84541248854",
    "source": "WEB"
  }

Response

status: Status response code.
response: Response content.
qualitativeAmount: Amount of qualitative (status) points pushed.
quantitativeAmount: Amount of quantitative points pushed.

Sample Response

Upon successful authentication, the API returns an access token.


  {
    "status": 200
    "response": "Amount was pushed successfully."
    "qualitativeAmount": "150"
    "quantitativeAmount": "45"
  }
