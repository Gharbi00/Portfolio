Push Notification Configuration

Introduction

To send push notifications to your customers, you need to configure your push notification settings. This section will guide you through the steps to set up push notifications for both web and mobile platforms within Elevok. You can find these settings in the Elevok Marketplace under the Notification app.

Execution Method

  • Elevok Execution: If you prefer Elevok to execute push notifications for you, simply configure the Elevok flag to enable this feature.
  • Custom Endpoint: If you wish to use your own push notification mechanism, prepare an endpoint to send push notifications.

Configuring Custom Endpoint

If you choose to use your own push notification mechanism, follow these steps:

  • URL: Provide the URL of your REST endpoint.
  • HTTP Method: Specify the HTTP method (e.g., POST) used to send push notifications.
  • Headers: Configure any required headers for authentication or other purposes.
  • Parameters: Define any parameters required by your endpoint, including the user key.
  • Notification Title Key: Specify the key used in the payload JSON to represent the notification title.
  • Notification Description Key: Specify the key used in the payload JSON to represent the notification description.
  • Notification Picture Key: Specify the key used in the payload JSON to represent the notification picture.
  • User Key: Specify the key used in the payload JSON to identify the recipient user.

Example Configuration

  • * URL: https://your-custom-endpoint.com/send-notification
  • * HTTP Method: POST
  • Headers
    • 
  Content-Type: application/json
  Authorization: Bearer your-auth-token
  • Parameters
    • 
  "rec_t": "tok-id",
  "api-f": "pif-65625"
  • Notification Title Key: "title"
  • Notification Description Key: "description"
  • Notification Picture Key: "picture_url"
  • User Key: "user_id"

Push Notification Execution

  • After configuring your push notification settings, Elevok will execute push notifications automatically if the Elevok flag is enabled.
  • If you are using a custom endpoint, Elevok will send push notification requests to your specified endpoint using the configured parameters.
  • Test your push notification setup to ensure that notifications are delivered as expected.

By properly configuring your push notification settings, you can effectively engage your customers across both web and mobile platforms. For more detailed instructions and troubleshooting tips, please refer to the Elevok Help Center.
