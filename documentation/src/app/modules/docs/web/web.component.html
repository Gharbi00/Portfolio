Elevok Widget Integration

Introduction

The Elevok Player Widget is a versatile interface that empowers your users with access to key loyalty program features directly on your website or web application. Follow this guide to seamlessly integrate the Elevok player's profile into your platform.

Embedding the Widget

To showcase the Elevok Player Widget on your website or web application, you'll need to insert a JavaScript code snippet into the relevant section of your webpage. Here's how:

  • Copy the Code Snippet: Grab the JavaScript code snippet provided below.
    • <!-- index.html -->
  
  <!DOCTYPE html>
  <html>
  <head>
    // Your head content ........
  
    <script src="https://widget.elevok.com/sdk.js" defer="defer"></script>
  </head>
  <body>
    // Your body content ........
  
    <script> window.widgetInit = { appId: 'Your_Client_Key', locale: 'en', theme: 'light' };</script>
  </body>
  </html>
  • Paste the Snippet: Insert the code snippet just before the closing body and the head tag in the HTML of your webpage. If you're utilizing dynamic page generation technologies such as PHP or ASP, include the snippet in the header file that is incorporated in each page.

Widget Modes

The Elevok Player Widget supports two modes: guest view and player view, serving distinct purposes while utilizing the same code snippet.

  • Guest View: Ideal for non-logged-in users to explore your loyalty program and encourage registration.
  • Player View: Tailored for logged-in users to track their progress and enjoy program benefits.

Player View Setup

To switch to player mode, execute the Elevok Consumer Login API during user login or registration in your main application. This API notifies the widget about the new user, enabling seamless transitions between guest and player views.

Upon initializing the widget with player data, Elevok's player profile undergoes creation or updating on the backend. To enrich your Elevok player profile with attributes not directly accessible via the user interface, send the dta you need via the Player Login API.
Selecting an Immutable Player Unique ID The Elevok user profile is established utilizing the playerUniqueId. It's imperative to opt for a unique identifier that remains constant and is never altered. Should this identifier change for a particular player, there's a significant risk of forfeiting all original data associated with that player, including accrued points and rewards within Elevok. Thus, it's strongly advised against utilizing mutable identifiers such as email addresses or mobile numbers, as they are subject to change by the user at any given time.
