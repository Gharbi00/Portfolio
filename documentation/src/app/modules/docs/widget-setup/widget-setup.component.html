Send Theme and Language to Widget

Introduction

This API allows clients to send a specified theme and language to the widget. It provides feedback on the success or failure of the operation.

Endpoint

URL: /widget/setup
Method: POST

Request Headers

Here you need to insert the bearer token obtained from the Login and Get Access Token API.


  {
    "Authorization": "Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  }

Request Parameters

reference*: Session reference found in local storate or cookies under key elvkwdigtref.
theme: The theme to be applied (either "dark" or "light").
language: The language code, must be one of the following: "en", "de", "fr", "ar", "ar_tn".

Note: theme and language parameters are optional. Minimum one of these parameter should be set.

Error Handling

500 Internal server error: In case of an error (HTTP 500), the client should ensure the request body is properly formatted, verify that the theme is either "dark" or "light," confirm the language code is one of "en," "de," "fr," "ar," or "ar_tn," and check the server logs for additional error information.

Sample Request

Upon successful authentication, the API returns an access token.


  POST /widget/setup
  Authorization: Bearer YOUR_ACCESS_TOKEN
  {
    "reference": "asdfa87asdf",
    "theme": "light",
    "language": "en"
  }

Sample Response

Upon successful request, the API returns a confirmation.


  {
    "success": true,
    "message": "Theme and/or language was sent successfully to widget."
  }
